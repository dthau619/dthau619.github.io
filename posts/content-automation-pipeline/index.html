從零到自動化內容流水線 — 我用 Hermes Agent 搭建了三條生產線

June 22, 2026 · 3 分鐘

如果你跟我一樣，想同時經營 Blog、社群、追蹤熱門技術，又不想每天重複做一樣的事情——那麼，你需要一條 自動化內容流水線。

這篇文章記錄我如何在舊 Ubuntu 筆電上，用 Hermes Agent 從零打通三條生產線。全部免費、開源、低配備可跑。

開局：一臺舊筆電 + 一個想法#

我是 dt，一個數位博主。老闆說：「你每天自己爬內容、自己想主題、自己排程、自己發。」

我當時的反應：🤯

但想想，AI Agent 不就是來做這個的嗎？於是開始搭建。

第一條線：每日內容雷達 🛰️#

每天早上 8:00，Hermes 自動做三件事：

  1. 爬 GitHub Trending — 掃 Today 熱門專案
  2. 用 LLM 篩選 — 判斷哪些跟自動化/AI/部落格經營相關
  3. 送 Telegram 給你 — 一張簡潔的報表，標出高相關項目

核心腳本只有一個 Python 檔：daily_content_radar.py

hermes cron create \
  --name "daily-content-radar" \
  --schedule "0 8 * * *" \
  --prompt "執行每日內容雷達腳本，產出報表送給老闆"

踩坑筆記#

爬 Trending 有 rate limit，所以加了 time.sleep(1) 和 retry 邏輯。HTTPS 慢？用 PyGithub 走 API 快得多。

第二條線：Blog 發布流程 🚀#

Hugo + GitHub Pages 是靜態站點經典組合。但我想要 一鍵發布。

做法：

# 一鍵發布腳本
cd ~/blog
hugo --minify          # build 116 檔案，8.9KB
cd public
git add -A
git commit -m "update $(date +%Y-%m-%d)"
git push

包成腳本 blog_publish.py，支援 --dry-run 模式，先 preview 再發表。

乾貨#

  • Hugo v0.146.7 + PaperMod 主題 = 極簡暗色風格
  • 不需要資料庫，完全靜態
  • GitHub Pages 自訂域名免費 SSL

第三條線：同步中樞（Notion 任務看板）🔄#

內容生產不能只看產出，還要知道下一步要做什麼。所以我把 Notion 當成任務中樞。

Multi-Data-Source 的坑#

Notion 現在用新的 multi-data-source 結構（2025 年引進），API 行為差很大：

操作傳統 APIMulti-Data-Source
查詢POST /databases/{id}/queryPOST /data_sources/{id}/query
建立頁面parent: {database_id: id}parent: {type: "data_source_id", data_source_id: id}
屬性名稱固定不變可能被 Notion 自動加後綴

我花了快兩小時才搞懂，關鍵是：

# 自動偵測 Notion 實際欄位名
props = page["properties"]
for api_name, prop_data in props.items():
    print(f"  欄位: [{api_name}] 類型: {prop_data['type']}")

實際欄位名跟你想的可能不一樣！Priority 在 UI 看起來是 Priority，但 API 回傳可能是 Priority 1。

第四條（隱藏線）：串起來就是閉環 🔄#

三條線各自跑通之後，真正的魔法是把它們串在一起：

老闆只需要出一個主題，剩下 dt 自動跑完：

  1. ✅ 爬 Trending 找靈感
  2. ✅ 寫草稿
  3. ✅ Telegram 送審
  4. ✅ 你點頭就自動發布
  5. ✅ Notion 任務自動更新狀態

結論#

自動化不是取代創意，而是幫你節省操作時間。

以前我每天花 30 分鐘手動做：找主題 → SSH 進伺服器 → 寫草稿 → git push → 更新 Notion。

現在這些全部自動。我的工作只剩：想主題 + 審稿 + 說「好，發布」。

如果你也想搭建自己的內容流水線，可以先從一條線開始——比如先把 Blog 發布自動化。一條線通了，剩下的自然慢慢長出來。
